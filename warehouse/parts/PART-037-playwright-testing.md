# PART-037: Playwright Testing & Ralph Loop

**Category:** POST_GEN | **Condition:** Every game (automated testing) | **Dependencies:** PART-011, PART-027, PART-035

---

## Purpose

Automated testing pipeline for generated games. After generating HTML (Stage 2), Playwright tests are generated from the game-specific template's test scenarios, then run in a fix loop until all tests pass and Claude approves the result.

## Architecture

```
ralph.sh orchestrates:

  ┌─────────────────────────────────┐
  │ 1. claude -p: Generate HTML     │ ← From game-specific template
  └──────────────┬──────────────────┘
                 ▼
  ┌─────────────────────────────────┐
  │ 2. claude -p: Generate tests    │ ← From template test scenarios + HTML
  │    (runs ONCE — tests are fixed)│    Output: tests/game.spec.js
  └──────────────┬──────────────────┘
                 ▼
  ┌─────────────────────────────────┐
  │ 3. npx playwright test          │◄──────┐
  └──────────────┬──────────────────┘       │
                 ▼                          │
           ┌───────────┐                    │
           │ Tests pass?│                    │
           └─────┬─────┘                    │
          yes    │    no                    │
           ▼     │     ▼                    │
  ┌────────────┐ │ ┌──────────────────┐     │
  │ 4. Claude  │ │ │ 4. claude -p:    │     │
  │ reviews vs │ │ │ Fix HTML based   │     │
  │ checklist  │ │ │ on test failures │     │
  └──────┬─────┘ │ └────────┬─────────┘     │
         ▼       │          └───────────────┘
  ┌────────────┐ │
  │ APPROVED?  │ │
  └──────┬─────┘ │
   yes   │   no  │
   ▼     │    ▼  │
  DONE   │  FIX──┘
         │
  (max 5 iterations)
```

## File Structure

```
game-spec/
├── testing/
│   ├── ralph.sh              ← The loop script (executable)
│   ├── playwright.config.js  ← Shared Playwright config (copied per game)
│   └── test-helpers.js       ← Shared test utilities (copied per game)
```

Per game output:
```
{game-directory}/
├── index.html
├── inputSchema.json
├── tests/
│   ├── playwright.config.js  ← Copied from testing/
│   ├── test-helpers.js       ← Copied from testing/
│   ├── game.spec.js          ← Generated by Claude (Step 2)
│   ├── test-results.json     ← Playwright output
│   └── test-output.txt       ← Console output
```

## Usage

```bash
# Full pipeline — generate HTML + tests + fix loop
cd game-spec
./testing/ralph.sh templates/queens/game templates/queens/spec.md

# Test existing HTML (skip generation)
SKIP_GENERATE=1 ./testing/ralph.sh templates/queens/game templates/queens/spec.md

# Test with existing tests (skip both generation steps)
SKIP_GENERATE=1 SKIP_TESTGEN=1 ./testing/ralph.sh templates/queens/game templates/queens/spec.md

# More iterations
MAX_ITERATIONS=10 ./testing/ralph.sh templates/queens/game templates/queens/spec.md
```

## Test Helpers (test-helpers.js)

Shared utilities available to all game tests:

| Helper | Purpose |
|--------|---------|
| `waitForGameReady(page, timeout)` | Wait for `gameState.isActive === true` |
| `collectConsoleErrors(page)` | Capture console errors during test |
| `getGameState(page)` | Get current gameState via `debugGame()` |
| `waitForGameComplete(page, timeout)` | Wait for `game_complete` postMessage |
| `sendGameInit(page, content)` | Send `game_init` postMessage to game |
| `assertContainedInLayout(page, selector)` | Verify element within 480px wrapper |
| `assertSingleActionButton(page)` | Verify only one of Submit/Retry/Next visible |
| `validateMetrics(metrics)` | Validate metrics against contract |
| `validateAttempt(attempt)` | Validate attempt against contract |
| `verifySentryIntegration(page)` | Verify full Sentry setup: config, SDK, init, script order, debug functions |

## Playwright Config

- **Viewport:** 480x800 (mobile, matches game max-width)
- **Browser:** Chromium headless
- **Web server:** `npx serve` on port 8787 (auto-started)
- **Reporter:** JSON (for machine-readable results) + list (for console)
- **Screenshots:** On failure only

## Test Categories (Mandatory)

Every generated `game.spec.js` must include tests for:

1. **Page Load** — no errors, globals defined, game state initialized
2. **Layout** — content inside `#gameContent`, grid within 480px, no overflow
3. **Game Flow** — rounds render, advance, complete
4. **Button States** — mutual exclusivity, correct show/hide
5. **Validation** — correct/incorrect answers, visual feedback, attempt recording
6. **Metrics** — accuracy, stars, postMessage shape
7. **Sentry Integration** — SentryConfig + SDK loaded, initialized, script order correct, debug functions exist (use `verifySentryIntegration` helper)
8. **Game-Specific** — unique mechanics of this game (≥3 tests)

## Key Design Decision: Tests Are Fixed

Tests are generated **once** (Step 2) and **never modified** by the fix loop. This prevents Claude from "fixing" failures by weakening tests. Only `index.html` is modified during the fix loop.

## Rules

- Tests MUST use the shared helpers from `test-helpers.js`
- Tests MUST simulate real user actions (`page.click()`, `page.fill()`) not `page.evaluate()`
- Tests MUST NOT test actual audio output — mock FeedbackManager calls
- Tests MUST be independent — each test can run in isolation
- The ralph loop MUST NOT modify `game.spec.js` during fix iterations
- Max iterations defaults to 5 — override with `MAX_ITERATIONS` env var
