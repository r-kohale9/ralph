# PART-035: Test Plan Generation

**Category:** POST_GEN | **Condition:** Every game (post-generation step) | **Dependencies:** PART-011, PART-027

---

## Purpose

After generating the game HTML, create both:
1. **`tests.md`** — Human-readable test plan documenting all testable behaviors
2. **`tests/game.spec.js`** — Playwright test file (generated by ralph loop, PART-037)

This part defines the test plan format. The Playwright test file is generated separately by the ralph loop using this plan + the generated HTML as input.

## When

This runs as a **post-generation step** — after Stage 2 produces the HTML file.

## Instructions for LLM

After generating `index.html`, analyze the game code and produce `{game-directory}/tests.md` covering ALL of the following categories.

## 1. Page Load & Initialization

- [ ] Page loads without console errors
- [ ] `waitForPackages()` resolves within 10 seconds
- [ ] `FeedbackManager.init()` completes successfully
- [ ] `setupGame()` runs and renders first round
- [ ] All package globals are defined: `FeedbackManager`, `TimerComponent`, `VisibilityTracker`
- [ ] Game uses fallback content when no postMessage received
- [ ] `gameState.isActive` is `true` after setup
- [ ] `gameState.startTime` is set to a valid timestamp

## 2. PostMessage Integration

- [ ] Game listens for `game_init` message
- [ ] Received content is stored in `gameState.content`
- [ ] Game re-renders with received content
- [ ] Invalid/malformed messages are ignored (no crash)
- [ ] `game_complete` message sent on game end with correct shape:
  - `type: 'game_complete'`
  - `data.metrics` contains `accuracy`, `time`, `stars`
  - `data.attempts` is an array
  - `data.events` is an array

## 3. Game Flow

### Round Progression
- [ ] First round displays correctly on load
- [ ] Round counter shows "1/{{totalRounds}}"
- [ ] After correct answer + next, round advances
- [ ] After all rounds complete, `endGame()` fires
- [ ] `gameState.currentRound` increments correctly

### User Interaction
{{For each interaction type in the game, add specific tests:}}
- [ ] {{Interaction 1}}: {{expected behavior}}
- [ ] {{Interaction 2}}: {{expected behavior}}
- [ ] Interaction is disabled after submission (prevent double-submit)
- [ ] Reset button clears all user input to initial state

## 4. Validation

### Correct Answer
- [ ] Correct answer is accepted
- [ ] Visual feedback shown (green highlight / correct class)
- [ ] Submit button hides, Next button appears
- [ ] Score increments
- [ ] Attempt recorded with `correct: true`

### Incorrect Answer
- [ ] Incorrect answer is rejected
- [ ] Visual feedback shown (red highlight / incorrect class)
- [ ] Submit button hides, Retry button appears
- [ ] Score does NOT increment
- [ ] Attempt recorded with `correct: false`

### Edge Cases
- [ ] Empty input is handled (submit disabled or shows error)
- [ ] Whitespace-only input is handled
- [ ] Very long input doesn't break layout
- [ ] Special characters don't cause errors
- [ ] Rapid repeated submissions don't cause double-counting

## 5. Button States

- [ ] Submit visible initially (or after first interaction)
- [ ] Only ONE of Submit/Retry/Next visible at a time
- [ ] Reset always visible during gameplay
- [ ] After correct: Submit hidden, Next visible
- [ ] After incorrect: Submit hidden, Retry visible
- [ ] After retry: Submit visible again
- [ ] All buttons disabled after endGame

## 6. Timer (if applicable)

- [ ] Timer displays on screen
- [ ] **Timer actually starts counting — not stuck at 00:00** (use `assertTimerStarts` helper)
- [ ] Timer counts down/up correctly
- [ ] Timer pauses when tab is hidden
- [ ] Timer resumes when tab is visible
- [ ] Timer `onEnd` triggers correct behavior (endGame or next round)
- [ ] `timer.destroy()` called in endGame

## 7. Visibility Tracker

- [ ] VisibilityTracker created with `onInactive` and `onResume`
- [ ] Tab switch triggers pause (timer pauses, audio stops)
- [ ] Tab return triggers resume (timer resumes)
- [ ] Pause popup appears on tab switch
- [ ] `duration_data.inActiveTime` records inactive periods
- [ ] `duration_data.totalInactiveTime` accumulates correctly

## 7b. Preview Screen

- [ ] Preview screen renders before game interaction starts
- [ ] Preview header bar shows back button, avatar, question label, score, star
- [ ] Progress bar animates from 100% to 0% over duration
- [ ] Skip button visible and advances to game when clicked
- [ ] Preview auto-advances when timer/audio ends
- [ ] Audio plays during preview via FeedbackManager (if configured)
- [ ] Avatar switches between speaking/silent video based on audio state
- [ ] `startGameAfterPreview()` called after preview ends
- [ ] `gameState.startTime` set AFTER preview ends (not during init)
- [ ] `duration_data.preview[]` populated with `{ duration }` entry
- [ ] VisibilityTracker pauses preview on tab switch, resumes on return

## 8. Metrics & Scoring

- [ ] `accuracy` = correct / total (0-100 range)
- [ ] `time` = elapsed time in seconds
- [ ] `stars` calculated correctly (see `contracts/metrics.schema.json` for defaults):
  - 3 stars: accuracy >= 80%
  - 2 stars: accuracy >= 50%
  - 1 star: accuracy > 0%
  - 0 stars: accuracy = 0%
- [ ] `attempts` array has one entry per user submission
- [ ] Each attempt has: `attempt_timestamp`, `time_since_start`, `input_of_user`, `correct`, `attempt_number`

## 9. Results Screen

- [ ] Results screen shown after endGame
- [ ] Game screen hidden when results shown
- [ ] Score displays correctly (e.g., "8/10")
- [ ] Time displays in readable format
- [ ] Star count matches calculated stars
- [ ] Accuracy percentage displays correctly

## 10. Layout & Containment

- [ ] All game HTML is inside `#gameContent` (when using ScreenLayout)
- [ ] Game fits within 480px max-width
- [ ] Grids have pixel max-width (e.g. 360px), not just `max-width: 100%`
- [ ] No horizontal scrolling
- [ ] Content vertically centered
- [ ] All text readable (no overflow/truncation)
- [ ] Buttons are tappable size (min 44px)
- [ ] **If header/progress bar is present, it must be at the very top of the screen — no margin, no gap above it** (use `assertHeaderAtTop` helper, verified with screenshot)

## 11. Debug Functions

- [ ] `window.debugGame()` returns game state object
- [ ] `window.debugAudio()` returns audio state
- [ ] `window.testAudio()` plays test sounds
- [ ] `window.testPause()` simulates pause
- [ ] `window.testResume()` simulates resume

## 12. Error Handling

- [ ] Package loading timeout shows error (doesn't hang forever)
- [ ] Missing content gracefully falls back
- [ ] All async operations wrapped in try/catch
- [ ] No unhandled promise rejections
- [ ] Console shows structured JSON logs (not raw objects)

## 13. CSS & Styling

- [ ] No hardcoded color values for game feedback
- [ ] Correct answer uses `var(--mathai-green)` / `var(--mathai-light-green)`
- [ ] Incorrect answer uses `var(--mathai-red)` / `var(--mathai-light-red)`
- [ ] Buttons use `var(--mathai-green)` (primary) / `var(--mathai-blue)` (secondary)
- [ ] Grid cells have `overflow: hidden` if using pseudo-elements
